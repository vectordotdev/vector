Added `authoritative` field to sink acknowledgements configuration. When a source has end-to-end acknowledgements enabled, only **authoritative** sinks block source acknowledgements. Non-authoritative sinks have their event finalizers stripped, allowing the source to acknowledge events as soon as all authoritative sinks have processed them. This prevents non-critical sinks (console, metrics, etc.) from blocking acknowledgement of critical sinks. The `authoritative` field defaults to the value of `acknowledgements.enabled`, preserving full backwards compatibility.
