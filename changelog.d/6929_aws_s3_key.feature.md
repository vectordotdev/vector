The `aws_s3` sink now supports an optional `key` field that lets you template the full S3 object key from event data (for example `key = "logs/{{ host }}/%F.log"`). When set, `key` is rendered as the complete object key and the existing `key_prefix`, `filename_time_format`, `filename_append_uuid`, and `filename_extension` fields are ignored. This makes it easier to migrate from Fluentd's `s3_object_key_format`. Uniqueness is the user's responsibility — include a high-cardinality field or fine-grained timestamp in the template to avoid overwriting objects.
