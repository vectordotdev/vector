Improved AWS S3 source error logging and diagnostics. S3 GetObject failures now include AWS error codes, HTTP status, and actionable remediation guidance. Non-retryable errors (e.g. NoSuchKey, AccessDenied) can now delete the SQS message when `delete_failed_message = true`, preventing infinite retry loops. Backoff retry logging promoted from `trace!` to `warn!`, and startup diagnostic logging now includes `client_concurrency`, `compression`, `multiline`, `delete_failed_message`, and `acknowledgements`.
