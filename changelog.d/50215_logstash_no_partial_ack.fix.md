Fixed the `logstash` source to ACK only completed writer windows rather than also emitting a partial ACK for a window that is only partially present in a batched read. The partial ACK carried a sequence number that corresponded to no window boundary the client declared, and when an intermediary (such as a load balancer or service mesh) buffered and later misdelivered it onto a different connection, the Beats/Filebeat client rejected it as `invalid sequence number received (seq=N, expected=M)` and reconnected. A window split across reads is now ACKed only once its final event arrives. The source also now rejects a `WindowSize` frame that arrives before the current window has received all its advertised events as a fatal decode error (closing the connection), matching the upstream Logstash server, which guarantees every window is either complete or a genuine trailing tail.
