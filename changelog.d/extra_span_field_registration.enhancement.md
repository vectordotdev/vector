Downstream crates can now extend the allowlist of tracing-span fields preserved on Vector's internal observability output without modifying Vector. Use the `register_extra_span_field!("my_field")` macro from `vector-core` (re-exported as `vector_lib::register_extra_span_field!`) at module scope; a single registration covers both channels: any internal metric emitted from inside a span carrying that field will pick it up as a metric label (via `VectorLabelFilter`), and `internal_logs` will capture it onto the log event under `vector.<field>` (via `SpanFields`). Vector's own built-in global metric labels (`component_id`, `component_type`, `component_kind`, `buffer_type`) now go through the same `MetricLabel` registry, snapshotted once into a `HashSet` on first use; the existing `component_*` prefix gate on the log side is unchanged.
