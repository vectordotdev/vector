GCP sinks and sources now support [Workload Identity Federation](https://cloud.google.com/iam/docs/workload-identity-federation) (`external_account` credentials), letting workloads running outside GCP (AWS, Azure, on-prem) authenticate to Google Cloud by exchanging OIDC or SAML tokens for short-lived access tokens — eliminating the need to distribute long-lived service account keys. Point `credentials_path` (or `GOOGLE_APPLICATION_CREDENTIALS`) at an `external_account` JSON file and Vector will use it automatically. Authorized-user and impersonated-service-account credential files are also newly accepted.
