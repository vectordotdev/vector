# Shelved properties — explored, not selected

20 disk-buffer properties evaluated in the 2026-06 pass but NOT selected into the grind-plan G0-G8 (`../grind-plan.md`) or `../experiment-spec.md`. Reference material for the research record, not a live plan. Each was shelved for one of: subsumed by a live oracle, descoped to in-tree `proptest`, a deterministic logic bug fit for a unit test, blocked on an unconfirmed fault capability (bit-flip, filesystem-tamper, binary-swap, panic-injection), or needing a test-binary knob (`MAX_FILE_ID=6`, injected file-id state) absent from the production binary Antithesis runs.

The #21683 wedge, stated once: `decrement_total_buffer_size` does a raw unsaturated `fetch_sub`; an over-decrement wraps the counter to ~2^64, `is_buffer_full()` stays permanently true, and the writer blocks forever in `ensure_ready_for_write`. Rows below name additional independent triggers for this same wedge.

Committed SUT-side DETECTORS (report to Antithesis, do not guard) under the `antithesis` feature in `lib/vector-buffers/src/variants/disk_v2/`: `get_total_records` underflow, `total_buffer_size` decrement underflow, reader size-delta underflow, reader record-id monotonicity `assert_unreachable`, `record_bytes > 8` no-empty-record, writer record-never-spans-files, plus anti-vacuity `assert_sometimes!` reachability points. Authoritative list in `../experiment-spec.md` G2 and `../existing-assertions.md`.

| corruption-skip-record-id-accounting-consistent (Safety, SUT-side) | A corruption roll causes no accounting corruption | Abandoned tail alone yields `bytes_read < metadata.len()` (SAFE partial-read); underflow needs `bytes_read > metadata.len()` i.e. a CONCURRENT truncation, so pure corruption-roll is NOT itself a #21683 trigger | Covered by the underflow proptest descope plus committed detectors |
| file-id-rollover-stays-coordinated (Safety, LATENT BUG) | Reader/writer file-id sync survives u16 wrap | "next" arithmetic is wrap-correct (`(id + 1) % MAX_FILE_ID`), but the `seek_to_next_record` sync gate is a raw non-wrap-aware `reader_file_id > writer_file_id` u16 compare: reader on 5, writer wrapped to 0 → `5 > 0` true → reader wrongly claims sync → premature break or unbounded non-terminating seek loop (livelock); `unacked_reader_file_id_offset` does not fix it; production `MAX_FILE_ID=65535` needs ~8 TB, `=6` and `unsafe_set_*` are `cfg(test)`-only | Needs a test-build or runtime knob to inject file-id state; unreachable organically |
| overflow-chain-no-unaccounted-gap (Safety) | `WhenFull::Overflow` loses no events on crash | A crash loses LATER in-memory overflow events while EARLIER disk events survive → MIDDLE gap, not the suffix gap dedup assumes; unbiased `tokio::select!` over base/overflow interleaves and reorders even in steady state; overflow send sets `was_dropped = true` and calls `increment_dropped_event_count_and_byte_size` though the item went to overflow not loss → misleading drop counter | Needs a second topology config + crash; not in the converged two-node block-mode experiment |
| graceful-shutdown-flushes-all (Liveness) | Graceful shutdown flushes everything → no loss | `BufferWriter::Drop` calls `close()` (marks done + notify) but never `flush()`/`sync_all()` (Drop sync, flush async), so bytes staged in the 256 KiB `TrackingBufWriter` at drop are lost; zero-loss rests on the per-record `flush().await` finishing before `inputs` is dropped in `RunningTopology::stop` plus OS page-cache flush at clean exit, neither guaranteed and both broken by OOM-kill after SIGTERM; loss bounds: SIGKILL ≤500 ms page cache, reload ≤256 KiB writer, graceful "claimed zero" | Steady-state sibling of the LIVE config-reload experiment exercising the same Drop-no-flush root via SIGHUP |
| throughput-progresses-under-contention (Liveness) | Parallel senders keep making progress under throttle | All senders share one `Arc<Mutex<BufferWriter>>`; under CPU throttle mutex hold (encode + copy + maybe flush) starves throughput toward zero without deadlocking ("degenerate-but-alive" state 2 the deadlock detector cannot see); GA notes ~90 MiB/s ceiling at 10-thread contention | Borders on perf-testing (Bias B1); floor threshold hard to calibrate without flakiness |
| ledger-corruption-no-sigbus-crashloop (Safety) | A corrupt `buffer.db` causes no SIGBUS crash loop | `buffer.db` held as live `memmap2::MmapMut` with NO SIGBUS handler repo-wide; `CheckBytes` runs only at init before mmap is live; (1) corruption before restart → clean `FailedToDeserialize` exit; (2) page-aligned zero-truncation → all-zero atomics pass `CheckBytes` → SILENT ledger reset re-delivering acked records; (3) truncation while RUNNING → next `get_archive_ref` hits an unmapped page → SIGBUS → death with no flush/log → supervisor re-maps → crash loop; guard: pre-access `len() >= LEDGER_LEN` check | Needs filesystem-truncation fault (unconfirmed); init-time variant testable by injecting a corrupt `buffer.db` at container start |
| buffer-survives-version-upgrade (Safety+Liveness) | On-disk layout stable across versions | `Record` has NO on-disk version field, only a do-not-touch comment; a layout change is read by rkyv zero-copy `archived_root` as garbage that can pass hand-written `CheckBytes` (types only) and a CRC over the new layout → silent corruption not a detected error; same for the unversioned mmap'd `LedgerState` | Needs a binary-swap / two-binary custom fault; low immediate value |

## Cross-cutting shelving reasons

- Underflow cluster → in-tree `proptest` (grind-plan pivot, 2026-06-02): accounting-underflow invariants are deterministically reproducible in-process, so the corruption-skip-accounting row and Cluster-A aliases lose Antithesis-specific value.
- Bias B1 — deterministic logic bugs (record-never-spans-files, throughput-under-contention, parts of graceful-shutdown) go to unit tests, no fault-search budget.
- Unconfirmed fault capabilities — bit-flip, filesystem-tamper, clock, panic-injection, binary-swap gate the corruption, ledger, version-upgrade, finalizer-panic, and lifecycle rows. The live plan uses only confirmed node-kill + SIGHUP reload.
- Test-binary knobs — file-id rollover needs `MAX_FILE_ID=6` or injected id state, unavailable in production-binary runs.
